# 三、基尼指数 gini_index

源码：

```python
def gini_index(groups):
```

作用：

```text
衡量当前划分后的混乱程度
```

---

## 基尼公式

```math
Gini = 1 - \sum p_k^2
```

其中：

- \(p_k\)：第k类样本占比

---

## 理解“纯”

例如：

```python
[1,1,1,1]
```

全是同一类：

```text
非常纯
```

基尼：

```math
1 - 1^2 = 0
```

---

如果：

```python
[1,1,0,0]
```

则：

```math
1 - 0.5^2 - 0.5^2 = 0.5
```

更混乱。

# 十四、predict

源码：

```python
def predict(node, row):
```

参数：

| 参数 | 含义 |
|---|---|
| node | 当前树节点 |
| row | 当前样本 |

---

# 十五、predict 的核心逻辑

```python
if row[node['index']] == node['value']:
```

真正含义：

```text
当前样本在该特征上的值
是否满足当前节点规则
```

---

例如：

```python
node =
{
    'index':1,
    'value':2
}
```

表示：

```text
检查：
root 是否等于 2
```

---

如果：

```python
row[1] == 2
```

成立：

```python
predict(node['left'], row)
```

否则：

```python
predict(node['right'], row)
```

---

# 十六、为什么进入 left/right

训练阶段：

```python
if row[index] == value:
    left.append(row)
else:
    right.append(row)
```

所以：

```text
满足条件 → left
不满足条件 → right
```

预测阶段必须一致。

---

# 十七、predict 的递归过程

第一次：

```python

```

即：

```python
node = tree（根节点）
```

---

之后：

```python
predict(node['left'], row)
```

于是：

```python
node = node['left']
```

不断深入。

# 十九、predict 本质

实际上：

```text
从根节点开始：

    检查当前规则

        满足：
            往左走

        不满足：
            往右走

重复
直到到达叶节点
```

# 二十一、为什么规则是 “==”

因为：

这个教学版代码使用的是：

```text
离散特征
```

例如：

```text
color:
0 青绿
1 乌黑
2 浅白
```

所以：

```text
是否等于某值
```

最自然。

---

而 sklearn 常见：

```text
<= 2.45
```

是因为：

```text
处理连续特征
```
